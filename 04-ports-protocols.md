# 04 — Common Protocols & Common Ports

---

## What are protocols?

Now, before we start talking about protocols and common ports, you might be thinking, "This already sounds complicated."

Trust me, don't worry.

Everything is going to make sense in a minute.

As we said before, protocols are simply a set of rules that every company follows.

Without them, networks would never look the way they do today.

Back then, if every company decided to build its own networking rules, everything would be a complete mess. Every company would follow its own standards, and the only way you and I could communicate would be if we were using devices from the exact same company.

That's exactly why protocols exist.

---

## Common Network Protocols

Now that we know what a protocol is, let's talk about some of the most common ones.

There are countless protocols out there, and every single one has its own purpose.

Some of the most common ones are:

* DNS
* FTP
* TCP
* UDP
* IP
* ICMP
* SNMP
* HTTP
* HTTPS
* POP3

...and many more.

Some protocols are mainly used for the web, like DNS, HTTP and HTTPS.

You might be wondering...

"So what's the difference between HTTP and HTTPS?"

Simple.

The **S** stands for **Secure**.

That means HTTPS encrypts the communication between you and the website to help keep your data protected.

Now let's talk about DNS.

Think of DNS like the old phone book we used to have.

Instead of remembering everyone's phone number, we'd simply look it up in the book.

DNS works almost the same way.

Without DNS, every time you wanted to visit Google, instead of typing:

`google.com`

you'd have to remember and type Google's IP address instead...

And trust me, that would be a nightmare.

We've already talked about TCP and UDP, so you already know what they do.

FTP is responsible for transferring files between computers and servers.

There are plenty of other protocols too, like IP and many others.

Each one has its own job and its own rules.

Think of protocols like laws.

Without laws, everything turns into chaos.

Networks work exactly the same way.

# Common Ports

So...

What are common ports?

These are the ports you'll run into all the time as a penetration tester.

You don't have to memorize every single one today, but you should definitely keep them written down somewhere you'll see often until they become second nature.

Think about it this way.

Your IP address is like your home's address.

The port is the apartment number.

In other words, ports make sure your data reaches the correct application.

For example...

When someone sends you a WhatsApp message, you want it to show up inside WhatsApp...

Not inside your file manager.

That's exactly what ports are for.

---

## Some Common Ports

| Port | Service | What it's used for                                           |
| ---- | ------- | ------------------------------------------------------------ |
| 80   | HTTP    | Unsecured websites                                           |
| 443  | HTTPS   | Secure websites                                              |
| 21   | FTP     | Transferring files between computers and servers             |
| 22   | SSH     | Secure remote access to servers and devices                  |
| 53   | DNS     | Translating domain names like `google.com` into IP addresses |
